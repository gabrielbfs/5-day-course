Agent Testing GUI

Test all endpoints of your agent locally

GET /health

Check if the agent is running and get status information

POST /query

Send a direct query to your agent. This is the standard query endpoint - no A2A routing.

Enter your question for the agent
Optional: User identifier (defaults to "anonymous")

POST /search

Automatically finds and routes to the best agent for your query. Uses LLM to select from the agentfacts database.

Describe what you want to do. The system will find the best agent automatically.
Optional: Conversation tracking ID
Optional: User identifier

POST /a2a

Agent-to-Agent communication. Must include @agent-id to route to another agent.

Include @agent-id to route to another agent (e.g., @furniture-expert Hello!)
Optional: Conversation tracking ID
Message role (default: "user")

GET /agents

List all known agents for A2A communication

GET /agentfacts

Get AgentFacts (NANDA Schema) - agent's identity, capabilities, and metadata

POST /agents/register

Register another agent for A2A communication

Unique identifier for the agent
Full URL to the agent's A2A endpoint
